# FJALLRAVEN MD分析用データベース 活用・分析ガイド

このドキュメントでは、ツールによって構築・蓄積されている `fjallraven_md_v3.db`（SQLiteデータベース）の構造と、Excelレポートの範囲を超えた**より高度なデータ分析**を行うための手法を解説します。

---

## 1. データベースの構造（テーブル定義）

データベースはデータの重複や無駄を省いた「**正規化**」された構造になっており、以下の4つのテーブルで構成されています。

### 🛒 `sales_raw` （今年度 売上トランザクション）

日々のレシート（明細）レベルの売上データです。

* **主なカラム**: `営業日付`, `店舗名称`, `商品コード`（D列相当）, `数量`, `税抜金額`, `客層区分`, `会員番号`
* **ポイント**: 商品名や原価などの「マスタ情報」は含まれていません。分析時は `master_data` と `商品コード` をキーにして結合（JOIN）します。

### 🛒 `sales_raw_prev` （前年度 売上トランザクション）

2025年などの前年売上データです。構造は `sales_raw` とほぼ同じです。

* **用途**: 前年比分析や、過去のトレンド調査に使用します。

### 📦 `stock_history` （在庫スナップショット）

ツールを実行した日ごとの在庫データが**蓄積**されます。

* **主なカラム**: `拠点名`, `商品コード`, `現在数量`, `受入数量`, `実行日`（ツールを実行した日付、YYYY-MM-DD）
* **ポイント**: 「過去の特定の日に、在庫がいくつあったか」を時系列で追うことができます。

### 📖 `master_data` （商品マスタ）

最新の1件だけが保持される商品カタログです。

* **主なカラム**: `商品コード`（A列相当）, `商品名`, `Brand`, `標準原価`, `商品種別1`（大分類）, `商品種別2`（中分類：Kanken等）, `商品種別3`（小分類）, `ColorName`, `SizeCode`, `初回投入日`, `Drop`

## 2. DBの中身を見る方法（ツールの紹介）

SQLite形式のデータベースは、以下のツールを使って簡単に開くことができます。

### A. DB Browser for SQLite（おすすめ・非エンジニア向け）

* **特徴**: 無料で最も簡単。Excel感覚でテーブルを開き、フィルタリングやCSVエクスポートが可能です。
* **使い方**: インストール後、`fjallraven_md_v3.db` をドラッグ＆ドロップするだけ。「データ閲覧」タブで中身を見たり、「SQL実行」タブでクエリを書くことができます。

### B. Excelの「Power Query」

* **特徴**: いつものExcel上でDBに接続し、ピボットテーブルに直接データを流し込めます。（※事前にSQLite用ODBCドライバのインストールが必要な場合があります）
* **使い方**: 「データ」タブ ＞ 「データの取得」 ＞ 「その他のデータソースから」 ＞ 「ODBC」などを選択。

### C. Python / Jupyter Notebook

* **特徴**: グラフ化や機械学習など、高度な分析をしたい場合に最適です。
* **使い方**: `sqlite3` ライブラリと `pandas` を使って数行で読み込めます。（下記4章参照）

## 3. 実務ですぐ使える！分析SQLレシピ

DB Browser等で「SQL実行」機能を使うと、欲しいデータだけを瞬時に抽出できます。

> [!TIP]
> 以下のSQLは、コピー＆ペーストしてそのまま実行可能です。

### レシピ1: 店舗別・中分類別の売上金額（今年）

マスタと結合して、任意の切り口で集計する最も基本的な形です。

```sql
SELECT 
    s.店舗名称,
    m.商品種別2 AS 中分類,
    SUM(s.数量) AS 販売数,
    SUM(s.税抜金額) AS 売上金額
FROM sales_raw s
LEFT JOIN master_data m ON s.商品コード = m.商品コード
GROUP BY s.店舗名称, m.商品種別2
ORDER BY s.店舗名称, 売上金額 DESC;
```

### レシピ2: Kankenの「カラー×店舗」の売れ筋ランキング

特定のキーワードで絞り込みたい場合に便利です。

```sql
SELECT 
    m.ColorName AS カラー,
    s.店舗名称,
    SUM(s.数量) AS 販売数
FROM sales_raw s
LEFT JOIN master_data m ON s.商品コード = m.商品コード
WHERE m.商品種別2 LIKE '%Kanken%'
GROUP BY m.ColorName, s.店舗名称
ORDER BY 販売数 DESC;
```

### レシピ3: 過去の在庫推移（時系列トレンド）を追う

特定の商品（例: Kanken Classicの特定カラー）の在庫が、どう減っていったかを確認します。

```sql
SELECT 
    実行日,
    拠点名,
    現在数量
FROM stock_history
WHERE 商品コード = '23510xxx'  -- ※確認したい品番に書き換える
ORDER BY 実行日 ASC, 拠点名;
```

## 4. さらに高度なMD分析アプローチ（Python・Notebook等）

蓄積されたデータを活用して、中長期的に取り組むべき分析のアイデアです。NotebookLM等にCSVとして抽出したデータを食べさせる際にも役立ちます。

### 1. 売上ピークと天候・イベントの相関分析

* **手法**: `sales_raw` の日別売上データと、気象庁の過去の気温・降水量データを結合します。
* **MDの示唆**: 「気温が何度を下回るとJacketが急激に動くか」を定量化し、来期の投入タイミング（初回投入日）やプロモーションの精緻化に活かします。

### 2. 併売分析（バスケット分析・アソシエーション分析）

* **手法**: `sales_raw` の「レシート番号」をキーにして、「一緒に買われやすい商品の組み合わせ」を分析します。
* **MDの示唆**: 「Kankenと一緒に買われている小物は何か？」が分かれば、店舗でのVMD（商品陳列）の近接配置や、セット割引キャンペーンの設計に直接繋がります。

### 3. 客層別（インバウンド・会員）のLTVとリピート分析

* **手法**: `sales_raw` の「会員番号」をキーにして、顧客一人あたりの購買頻度や単価推移を追います。また、インバウンドフラグを使って「訪日客に刺さる特定のカラーやサイズ」を抽出します。
* **MDの示唆**: インバウンド需要に依存しているSKUのリスクヘッジや、国内優良会員向けのロイヤリティ向上施策の立案に役立ちます。

### 4. 滞留在庫（デッドストック）の早期検知アラート

* **手法**: `stock_history` の時系列データから、「在庫があるのに〇週間全く売れていない（販売数ゼロ）」商品を抽出します。
* **MDの示唆**: 単なる消化率だけでなく「動きの止まったタイミング」を検知し、マークダウン（値下げ）や店舗間移動をプロアクティブに提案できるようになります。

> [!NOTE]
> 自動生成されるExcelレポートは「毎日の状況をパッと見てアクションを決める（戦術）」ためのダッシュボードです。一方、DBを活用した分析は「来季の発注計画やブランド全体の戦略を決める（戦略）」ための武器になります。まずは「DB Browser for SQLite」等でデータに触れてみることをお勧めします。
