# MD本部用ダッシュボード 画面仕様書

> 対象: MD担当・バイヤー・本部スタッフ  
> 目的: 複数店舗横断での在庫最適化・売上戦略の意思決定を支援する  
> 更新タイミング: 週次（月曜AM） / 月次サマリーは月初

---

## 1. 画面構成（タブ切替方式）

```
┌──────────────────────────────────────────────────────────┐
│ ヘッダー: ブランド名 / 集計期間 / 対象店舗数             │
├──────────────────────────────────────────────────────────┤
│ [タブ] 全体サマリー | 在庫ヘルス | ABC/XYZ | 発注アラート  │
├──────────────────────────────────────────────────────────┤
│
│  【タブ①: 全体サマリー】
│    - 全店合計 売上/客数/客単価（前週比）
│    - 中分類別 売上構成比 横棒グラフ
│    - 前年同月比 テーブル
│
│  【タブ②: 在庫ヘルス推移】  ← 課題A対応
│    - 週次WOS推移グラフ（中分類別 折れ線）
│    - 在庫金額残高の推移（棒グラフ）
│    - 在庫回転率（IT）月次推移
│
│  【タブ③: ABC/XYZ分析】
│    - 9マトリクス ヒートマップ
│    - 戦略ガイドテキスト
│    - 販売機会損失額ランキング  ← 新規追加
│
│  【タブ④: 発注アラート】  ← 課題B対応 (V4.1)
│    - 需要予測値（来月）
│    - 推奨発注数
│    - 欠品リスクSKU一覧
│
└──────────────────────────────────────────────────────────┘
```

## 2. タブ別 表示項目定義

### タブ①: 全体サマリー

| 表示項目 | データソース | 備考 |
|----------|--------------|------|
| 全店合計 KPI（売上/客数/客単価） | sales_raw | 前週比あり |
| インバウンド比率 | sales_raw | 外国人フラグ集計 |
| 会員比率 | sales_raw | 会員フラグ集計 |
| 中分類別 売上構成比 | analyzer.py | 横棒グラフ |
| 前年同月比 テーブル | sales_raw + 前年ファイル | 月×中分類クロス |

### タブ②: 在庫ヘルス推移（新規）

| 表示項目 | データソース | 期間 |
|----------|--------------|------|
| WOS推移 折れ線グラフ | stock_history テーブル | 直近8週 |
| 在庫金額残高 棒グラフ | stock_history テーブル | 直近8週 |
| 在庫回転率（IT）月次 | sales_raw + stock_history | 直近3ヶ月 |
| WOS赤ライン | config.json: wos_red_below | 固定値 |
| WOS黄ライン | config.json: wos_yellow_above | 固定値 |

> **実装メモ**: `db_manager.py` に `get_stock_trend(weeks=8)` 関数を追加し、
> `excel_writer.py` の `write_stock_health_trend()` シートから呼び出す

### タブ③: ABC/XYZ + 販売機会損失

| 表示項目 | データソース | 備考 |
|----------|--------------|------|
| ABC_XYZ 9マトリクス | analyzer.py 既存 | ヒートマップ維持 |
| 戦略ガイド | 既存テキスト | 変更なし |
| 販売機会損失額ランキング | calc_opportunity_loss() 新規 | サイズ欠品SKU上位20 |

**販売機会損失額の表示カラム:**
- 商品コード / 商品名 / 欠品サイズ / 推定機会損失日数 / 推定損失額（円）

### タブ④: 発注アラート（V4.1以降）

| 表示項目 | データソース | 備考 |
|----------|--------------|------|
| 来月需要予測値 | forecaster.py 新規 | 乗法型季節指数法 |
| 推奨発注数 | forecaster.py | 安全在庫込み |
| 欠品リスクSKU一覧 | WOS < 閾値 AND 予測需要↑ | 優先度順ソート |
| 値下げシミュレーション | calc_markdown_simulation() 新規 | 粗利影響付き |

## 3. 複数ブランド対応方針（課題C）

`config.json` の変更案:
```json
// 変更前
"target_brand": "FRV"

// 変更後
"target_brands": ["FRV", "BAG"],
"brand_display_names": {
    "FRV": "Fjällräven",
    "BAG": "Bag Brand"
}
```

`main.py` での実装イメージ:
```python
for brand in config["target_brands"]:
    result = run_analysis(brand, config)
    save_report(brand, result)
```
