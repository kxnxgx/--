# 店舗用KPIボード 画面仕様書

> 対象: 店長・店舗スタッフ  
> 目的: 週次の売上・在庫状況を「ひと目で」把握し、当日の接客・補充アクションに直結させる  
> 更新タイミング: 週次（月曜AM）

---

## 1. 画面構成

```
┌──────────────────────────────────────────────────────┐
│ ヘッダー: 店舗名 / 集計期間（例: 2026/05/11〜05/17）  │
├──────────────────────────────────────────────────────┤
│ 【上段】主要KPI 5カード（横並び）                     │
│  売上 / 客数 / 客単価 / 1品単価 / 1客個数            │
├────────────────────────────┬─────────────────────────┤
│ 【中段左】属性KPI 2カード  │ 【中段右】WOSアラート    │
│  インバウンド比率 / 会員比率│  欠品危険件数 / 過剰件数  │
├────────────────────────────┴─────────────────────────┤
│ 【下段】Kanken在庫ステータス（カラー別・在庫0を赤表示）│
└──────────────────────────────────────────────────────┘
```

---

## 2. 表示項目定義

### 2-1. 主要KPIカード（5枚）

| # | 表示名 | 単位 | 比較基準 | 色付けルール |
|---|--------|------|----------|--------------|
| 1 | 売上 | 円 | 前週比（%） | プラス→緑↑ / マイナス→赤↓ |
| 2 | 客数 | 人 | 前週比（%） | プラス→緑↑ / マイナス→赤↓ |
| 3 | 客単価 | 円 | 前週比（%） | プラス→緑↑ / マイナス→赤↓ |
| 4 | 1品単価 | 円 | 前週比（%） | プラス→緑↑ / マイナス→赤↓ |
| 5 | 1客個数 | 個 | 前週比（%） | プラス→緑↑ / マイナス→赤↓ |

**各カードの表示要素:**
- 当週値（大きく表示）
- 前週比 ±%（矢印アイコン付き）
- 先週実績（小さく補足）

### 2-2. 属性KPIカード（2枚）

| # | 表示名 | 単位 | 比較基準 |
|---|--------|------|----------|
| 1 | インバウンド比率 | % | 前週比 pt差 |
| 2 | 会員比率 | % | 前週比 pt差 |

### 2-3. WOSアラートカード

| アラート種別 | 条件 | 表示色 | 表示内容 |
|------------|------|--------|----------|
| 欠品危険 | WOS < 2.0週 | 赤 | 該当SKU件数 + 商品名リスト上位5件 |
| 過剰在庫 | WOS > 20.0週 | 黄 | 該当SKU件数 |

> ※ 閾値は `config.json` の `wos_red_below` / `wos_yellow_above` に準拠

### 2-4. Kanken在庫ステータス

| 表示項目 | 内容 |
|----------|------|
| カラー名 | config.json の kanken_colors リストに準拠 |
| 在庫数 | 当週スナップショット |
| 状態表示 | 在庫0 → 赤背景＋⚠アイコン / 在庫あり → グレー背景 |

---

## 3. 非表示にするもの（店舗用は意図的に除外）

- ABC_XYZ分析
- 前年同月比（YoY）詳細
- カテゴリー構成比グラフ
- 消化率・シーズン分析
- 店舗横断比較（店舗スタッフには不要）
- 需要予測・発注推奨数（V4.1以降、MD担当のみ表示）

---

## 4. UX要件

- **スクロールなしで全KPIが見える**（1画面完結）
- **数字を読まなくても色と矢印で良し悪しがわかる**
- 印刷時にA4横1枚に収まること（店頭掲示を想定）
- ノンエンジニアが開くだけで使える（ファイルをダブルクリックするだけ）
