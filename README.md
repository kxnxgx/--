# MD分析ツール V3

本ツールは、店舗の売上データ・在庫データ・商品マスタを統合し、在庫最適化および販売戦略（MD）に直結する分析レポートを全自動で生成するPythonプログラムです。

## 🚀 使い方

1. **データの準備**
   以下のCSVファイルを `c:\分析` フォルダ（または基準フォルダ）内に配置してください。
   * `営業日別商品売上分析*.csv` (売上実績データ)
   * `在庫一覧*.csv` (現在の在庫データ)
   * `商品マスタ*.csv` (商品情報・原価データ)
   * （オプション）前年の売上データ（ファイル名に `2025` 等の前年キーワードが含まれるもの）
   ※ファイル名に日付等が含まれていても自動で最新のものを読み込みます。

2. **ツールの実行**
   フォルダ内にある **`実行ボタン.bat`** をダブルクリックしてください。
   黒い画面（コマンドプロンプト）が立ち上がり、処理が開始されます。

3. **レポートの確認**
   処理が完了すると、`v3` フォルダ内に **`MD分析レポートv3_YYYYMMDD.xlsx`** が生成されます。

## 📊 出力されるレポート（各シートの解説）

生成されるExcelファイルには、実務ですぐに使えるように最適化された複数のシートが含まれています。

### 1. 総合サマリー

* **用途**: ブランド全体の健康状態の把握。
* **内容**: 全体の売上個数、客数、客単価、セット率、インバウンド比率、会員比率などを一覧表示します。

### 2. カテゴリー構成比

* **用途**: 在庫の過不足のチェック。
* **内容**: 「中分類」ごとの「売上構成比」と「在庫構成比」を比較します。
* **可視化**: パーセント（%）表示の横棒グラフにより、売上のシェアに対して在庫を持ちすぎていないかが一目で判別できます。

### 3. 直近1ヶ月トレンド（1M分析）

* **用途**: 天候やキャンペーンによる直近のトレンド変化の把握。
* **内容**: 過去30日間の日次売上推移を中分類ごとに集計。
* **可視化**: ヒートマップ（カラースケール）で山と谷を視覚化。

### 4. ABC_XYZ分析（最重要シート）

* **用途**: SKU単位での発注・撤退・在庫確保の戦略決定。
* **内容**: 売上規模の「ABC分析」と、需要の安定性の「XYZ分析」を掛け合わせた統合評価。
* **可視化**:
  * AX（緑）〜CZ（赤）の9パターンの戦略ガイドを常時表示。
  * 回転率のデータバー表示。

### 5. 売上在庫詳細(WOS昇順)

* **用途**: 現場でのピンポイントな在庫補充や実績確認。
* **内容**: SKUごとの店舗名・カラー・サイズを含む詳細な売上と在庫リスト。消化率なども結合。
* **可視化**: **WOS（在庫週数）のヒートマップ**。2週未満は赤、20週超は黄色で警告表示されます。

### 6. 店舗別比較

* **用途**: 各店舗のカテゴリー別販売傾向の把握。
* **内容**: 店舗×中分類のクロス集計。店舗コード（数字→英数字順）によるソートが適用されています。

### 7. 消化率・シーズン分析

* **用途**: 品番ごとのシーズン状況と消化進捗の確認。
* **内容**: 受入数量に対する販売数の割合（消化率）。
* **可視化**: 消化率に応じた3色カラースケール。

### 8. Kanken詳細

* **用途**: 主力商品であるKankenシリーズの深掘り。
* **内容**: カラー別販売数と、店舗×サイズのクロス集計。

### 9. インバウンド分析

* **用途**: 外国人観光客向け商品の動向把握。
* **内容**: インバウンド売上に絞った中分類別の構成比。

### 10. 前年同月比

* **用途**: 前年実績との成長率比較。
* **内容**: 月別×中分類で今年と前年の販売数・売上金額を比較。

## ⚙️ システム構成と拡張性 (V3アーキテクチャ)

* **プログラム本体**: `v3\main.py` および各種モジュール (`data_loader.py`, `analyzer.py`, `db_manager.py`, `excel_writer.py`)
* **設定ファイル**: `v3\config.json`（カラム指定や閾値の変更が可能）
* **データベース**: `v3\fjallraven_md_v3.db` (SQLite)
  * 日々の売上・在庫のスナップショットが蓄積される設計。
  * 実行前に自動で `v3\backup` フォルダにバックアップを作成します（デフォルト7日間保持）。
