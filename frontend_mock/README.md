# 📊 店舗用KPIダッシュボード (HTMLモック ＆ 実データ連携)

本フォルダは、MD分析ツール V4 で追加された **HTML版店舗用KPIダッシュボード** のフロントエンドモックおよび連携ガイドです。
Pythonによるバッチ処理で生成された最新のブランド別JSONデータを読み込み、店舗ごとの主要KPI（売上、客数、客単価、セット率、Kanken在庫等）をリアルタイムにビジュアル可視化します。

---

## 🚀 使い方（実データ連携の起動手順）

以下の手順で、バッチ処理で生成された実データ（例: `dashboard_data_FRV.json`）をダッシュボードに反映してブラウザでプレビューできます。

### 1. 分析バッチの実行
あらかじめ `実行ボタン.bat` をダブルクリック（または `python v3/main.py` を実行）して、最新の分析データを生成しておきます。
（`v3/dashboard_data_[ブランド名].json` が自動出力されます）

### 2. APIサーバーの起動
ダッシュボードがローカルのJSONデータを読み込めるようにするため、軽量APIサーバーを起動します。
1. コマンドプロンプトまたはPowerShellで、本ツールのルートフォルダ (`c:\分析`) を開きます。
2. 以下のコマンドを実行します：
   ```bash
   python v3/server.py
   ```
3. `[Server] Running on http://localhost:8000 ...` と表示されれば起動完了です。

### 3. ブラウザでダッシュボードを表示
1. 本フォルダにある `store_kpi_dashboard.html` をブラウザ（Chrome, Edge, Safariなど）でダブルクリック等で開きます。
2. 画面上部にある「店舗選択」のプルダウンから、分析対象の店舗（例: 「FJALLRAVEN by 3NITY 心斎橋パルコ」など）を選択します。
3. 選択した店舗の売上実績、客数、客単価、および「Kankenカラー別在庫状況（在庫あり・欠品表示）」が瞬時に画面に反映されます！

---

## ⚙️ モックモードと実データモードの切り替え

`store_kpi_dashboard.html` のJavaScriptコード内には、テスト用モックデータと実データを切り替えるフラグが用意されています。

* **実データモード (推奨・デフォルト)**:
  `store_kpi_dashboard.html` 内の `const USE_MOCK = false;` となっている場合、起動したローカルAPIサーバー (`http://localhost:8000`) から動的にJSONを取得します。
* **スタンドアロン・モックモード**:
  APIサーバーを起動せずにHTMLファイル単体でデザインや挙動を確認したい場合は、`const USE_MOCK = true;` に書き換えてください。あらかじめ埋め込まれたデモ用モックデータで動作します。

## 💡 Kankenカラー別在庫ステータスの仕組み

本ダッシュボードの主要機能として、主力商品「Kanken (品番 23510)」の店舗別・カラー別在庫状況が動的に表示されます。
* **データ生成**: バッチ処理時に在庫CSVから該当店舗の「Kanken（23510から始まる品番）」の在庫を色ごとに自動集計し、JSONファイルに埋め込みます。
* **動的描画**: ダッシュボード上では、在庫があるカラーは「在庫（緑色）」として在庫数が表示され、在庫が0個のカラーは「欠品（赤色・⚠️警告マーク付き）」として表示されます。店頭での欠品対策や、他店からの取り寄せ・取り置きの判断に直結する設計となっています。
